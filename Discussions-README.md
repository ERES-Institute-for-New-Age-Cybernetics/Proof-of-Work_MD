## Discussion Categories

Discussions are organized around the ERES Trilogy's **BEST / SOUND / GOOD** architecture:

### BEST — Personal Graceful Evolution

Topics related to Book 1 instruments: UBIMIA economics, Meritcoin Proof-of-Resonance, EarnedPath credentialing (EP = CPM × WBS + PERT), PlayNAC engagement, 401(k) Soft Landing, Vacationomics.

*Questions to bring here:* How does UBIMIA differ from UBI? How does EarnedPath credentialing work? What is Proof-of-Resonance?

### SOUND — Public-Private Graceful Evolution

Topics related to Book 2 instruments: IDIPITIS identity architecture, NBERS economic security, GSSG governance, NPR (National Priorities Realignment), SCALULAR certification pillars (HEALTH/LAW/PROTECTION/SKILLS-TRADE), SROC contracts, Storm Party, ILO land stewardship.

*Questions to bring here:* How does SCALULAR certification work? What is the SCALULAR Engine's hidden trigger? How does GSSG differ from existing governance models?

### GOOD — Graceful Evolution Verified

Topics related to Book 3 instruments: FAVORS data integrity, CBGMODD stateful taxonomy, GAIA-SOMT infrastructure, BERA resonance metrics (ARI/ERI/RHC/RCI), ESVRD propulsion, FDRV vessel design, VLSA scale proof, MIEVM epistemology, SPT Papers.

*Questions to bring here:* How does MIEVM validation work? What is the ESVRD Phase 1 experiment? How do BERA metrics feed into Meritcoin?

### Cross-Cutting

Topics spanning multiple books or addressing the framework as a whole: C = R × P / M equation set, Triune Math, three eras (CyberRAVE/SaleBuilders/ERES), CCAL v2.1 licensing, ResearchGate publications, collaboration proposals, publishing strategy.

### Code & Development

Technical discussions about the PlayNAC KERNEL codebase, BERA-PY library, VERTECA voice navigation, GitHub Actions workflows, and repository architecture.

## Participation Guidelines

### The ERES Ethical Constraint

All discussion participants are expected to operate within the framework's foundational principle:

> *"Don't hurt yourself. Don't hurt others. Build for generations to come."*

This is not a slogan — it is the moderation standard. Contributions that violate this constraint will be addressed.

### Constructive Engagement

The ERES framework is morally vectored toward BEST/SOUND/GOOD end-outcomes. Discussion should reflect this: critique the architecture, challenge the equations, question the deployment assumptions — but do so in service of making the work stronger, not in service of tearing it down.

### Canonical Terminology

When discussing ERES frameworks, please use canonical terminology. Common corrections:

| Canonical | Not This |
|-----------|----------|
| SOMT = Sociocratic Overlay Metadata Tapestry | ~~Strategic Optimization & Merit Tracking~~ |
| SROC = Smart-Resonant Offset Contracts | ~~Smart Registered Offset Contracts~~ |
| UBIMIA = Universal Basic Infrastructure, Meritcoin Incentivized Architecture | ~~Universal Basic Income + Merit + Incentives + Awards~~ |
| RHC = Resonant Harmony Cycle | ~~Resonant Harmony Cybernetics~~ |
| Proof-of-Resonance | ~~Proof-of-Work (in Meritcoin context)~~ |

### How to Participate

1. Navigate to the [Discussions tab](https://github.com/ERES-Institute-for-New-Age-Cybernetics/Discussions/discussions)
2. Browse existing threads or start a new discussion
3. Tag your discussion with the appropriate category (BEST / SOUND / GOOD / Cross-Cutting / Code)
4. Reference specific documents or repos when possible
