# External-Constraint Test (SCALULAR-classified) — Audit Ledger v1.0

**Test (canon form):** A boundary, symmetry, or equivalence claim PASSES only if it constrains a *family* of objects via a structural property *external to* the claim's own definition. If the claim is recovered just by unpacking the definition of the quantity it's about, it FAILS — it's a restatement wearing a name, not a finding.

**Scope of this pass:** every claim explicitly flagged, proposed, or tested across this session and the directly-linked prior sessions. Not an exhaustive sweep of the full ~2,078-instrument archive — flag if you want a deeper pass against a named instrument set.

---

**Net read:** the test cleanly separates two classes that look superficially similar in this corpus — derivations using an external structural fact (group closure, divergence behavior, geometric universality) vs. constructions where two independently-chosen numbers or labels are pointed at each other and called convergence. Five pass, four fail, two flagged. The fails cluster on one signature every time: numerology/labeling without an external constraint doing the work.

## Recursive Validity Check (self-application)

The test can be applied to itself, using the deduction/induction cross rather than self-assertion:

| Arm | Path | Content |
|---|---|---|
| Deduction | Theory → Prediction | Theory = the analytic/synthetic distinction (standard, ERES-external epistemology). Prediction = the test's specific rule ("must constrain via an external structural property"). |
| Induction | Data → Inference | Data = the nine audited cases above, sorted independently. Inference = the pattern recovered from them: fails share an independently-constructed-numbers/labels signature; passes share a derivation-from-external-invariant signature. |

**Result:** the two arms converge on the same rule from independent directions — the test passes itself.

**Standing condition for any future recursive pass:** this only holds because the *Theory* node sits outside the ERES corpus (general epistemology, not an ERES-internal claim). If a future self-check used an ERES-internal claim as the Theory node, the check would fail by its own rule — two internally-constructed things validating each other is exactly the JAS×EMA=72≡36×2 failure mode, one level up. Any recursive application must re-verify this condition before counting as a pass.
