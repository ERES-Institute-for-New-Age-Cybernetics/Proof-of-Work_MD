## §0.3 The Three Intake Operations: Qualify · Quantify · Rationalize

When a Civigen submits a query, it enters the Instrument through three intake operations performed in sequence with explicit return values. Under the Pre-Qualification Inversion (§0.14, Lock Box C), the User-GROUP's selection profile is established **before** the round-trip opens; the three intake operations run *within* the selection profile.

### Qualify — Admissibility Gating (SEPLTA, Administrative)

The qualify stage answers: *is this query admissible within the User-GROUP's selection profile?* SEPLTA — **Social · Ecological · Political · Legal · Technical · Administrative** — is the six-domain admissibility frame. The Administrative domain handles round-trip administration itself: record-keeping, audit, attestation interface, DOFA governance touch-points.

A query that fits one or more domains and falls within the User-GROUP's selection profile passes the qualify gate with a domain tag. Tag is consequential: it determines BERA measurements (quantify), activated cells (rationalize), and selected-cell Trilogy filter passes.

The qualify stage is conservative; uncertain admissions return VEILED-A.

### Quantify — BERA Measurement Mapping

The quantify stage answers: *what BERA measurements does this query require, within the selection profile?* BERA produces RATE across R₁–R₇. The User-GROUP's selection profile may constrain which dimensions are computed.

The quantify stage produces a **BERA mask** — specification of which RATE dimensions the round-trip computes, intersected with selection-profile permissions. The mask determines DATABANK fields queried, Trilogy filter passes activated, and Sim OUTPUT fields populated.

Conservative; un-determinable masks return VEILED with scope-specification request.

### Rationalize — Honoring the Selection (Not Performing the Selection)

The rationalize stage answers: *given the User-GROUP's selection profile, which selected cells does this query activate?* Under Lock Box C, the rationalize stage **does not select cells.** Cells are selected by the User-GROUP before round-trip opens (and trinity-vetted by the Waiting Room/Greenbox per §0.14). The rationalize stage determines which already-selected cells the current query activates.

A query under selection profile `{One-Good × HELP, Data-Integrity × LAW}` activates at most those two cells. Query content outside the profile produces VEILED with "query falls outside selection profile" diagnosis and recommendation to widen the profile (which requires re-pre-qualification, not an in-round-trip operation).

The rationalize stage is bounded by the selection profile. This bounding is the architectural inversion's load-bearing consequence.
