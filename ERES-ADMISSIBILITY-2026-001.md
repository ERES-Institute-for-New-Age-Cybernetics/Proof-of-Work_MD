# **VII. Repeatable Drift: The Named Failure Mode**

The Jennings-Bates challenge names a real failure mode. ERES names it repeatable drift: a system whose outputs are stable while its internal transition logic drifts. Repeatable drift is the architectural analogue of financial fraud that produces consistent quarterly reports while the underlying positions deteriorate.

The ERES architecture treats repeatable drift as a primary failure mode rather than an edge case because:

It is the failure mode most likely to survive casual inspection. Consistent outputs are what evaluators check first. An engine optimized to produce consistent outputs while running on unstable internals will pass surface-level evaluation indefinitely.

It is the failure mode that accumulates. Each consistent output adds weight to the trust position. When the underlying transition logic eventually produces an inconsistent output, the accumulated trust position amplifies the damage.
