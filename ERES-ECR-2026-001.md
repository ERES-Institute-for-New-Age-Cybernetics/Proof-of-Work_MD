## **Abstract**

The Erasure Completeness Ratio (ECR) is the empirical falsifiability metric of ERES governance. It measures, for any given operational window, the fraction of attestations, permissions, claims, or system states that should have been revoked or retired and that actually were revoked or retired — across every downstream surface that ever held them. ECR emerges directly from the MPAM↔MIEVM Duality: every affirmative attestation has a mirror erasure obligation, and the integrity of the affirmative side is only as strong as the closure of the erasure side. This whitepaper defines ECR formally, anchors it within the canonical Triune Math root C = R × P ÷ M, specifies its instrumentation through EAAP and EAAP-CRL under ML-DSA/Falcon post-quantum signatures, and demonstrates why ECR (not narrative coherence, not symbolic elegance) is the metric that converts ERES from an abstract narrative system into a governable, empirically testable control law over a one-thousand-year horizon.

# **4\. Erasure Completeness Ratio — Formal Definition**

## **4.1 The ratio**

For an operational window W and a population of governance objects O (attestations, permissions, claims, states, credentials):

**ECR(W, O) \= | { o ∈ O : o.required\_erasure(W) ∧ o.actually\_erased(W) } |**

**÷  | { o ∈ O : o.required\_erasure(W) } |**

ECR is bounded in \[0, 1\]. By construction:

* **ECR \= 1.0** — every object that should have been erased, was erased, everywhere it propagated.

* **ECR \< 1.0** — residual entries persist: stale signatures still verifying, revoked permissions still being honored, retracted claims still propagating downstream. The complement (1 − ECR) is the leakage fraction.

* **ECR \= VEILED** — reserved state for windows in which the required-erasure population is itself unresolvable (the denominator cannot be determined). VEILED replaces the earlier ⊥ (bottom) notation; it is not zero, not undefined, and not failure — it is the explicit acknowledgement that the question is not currently answerable, which itself is information.

## **4.2 The seven RATE dimensions**

ECR is not a single scalar in production deployments. It is computed across the seven RATE dimensions yielded by the canonical relation (CBGMODD × FAVORS) \+ BERA \= RATE, indexed R₁ through R₇. A deployment reports a vector ⟨ECR₁, ECR₂, …, ECR₇⟩, and the convergent mean across the seven is what is compared to the MIEVM GO threshold. A scalar reported without the seven-vector behind it is presumptively VEILED.

## **4.3 Why this is empirically testable**

Each term on the right-hand side is operationally defined and externally measurable:

* required\_erasure(W) is determined by the COI charter and the EAAP-CRL revocation list — both public, both signed, both auditable.

* actually\_erased(W) is determined by polling every downstream surface (registries, mirrors, cached attestations, replica nodes) and confirming that signatures over revoked objects no longer verify.

* The denominator is finite and enumerable within W; the numerator is a subset of the denominator by construction.

This is the property that distinguishes ECR from a narrative claim. A narrative claim that “the system retires obsolete permissions” cannot be falsified. ECR can be falsified, in principle, by any third party with access to the EAAP-CRL and the downstream surfaces; if their independent computation diverges from the ERES-reported ECR, the discrepancy is itself a measurable error term.

# **6\. ECR as the Falsifiability Hook**

The provocation in Section 1 reduces, formally, to: “provide an operationally defined quantity Q such that an external party can compute Q′ from the same inputs and compare Q′ to ERES-reported Q.” ECR is that quantity.

## **6.1 The independent re-derivation test**

For any window W, an external auditor with access to:

* the published EAAP-CRL for W,

* the set of downstream surfaces declared in the COI charter,

* the post-quantum verification keys (ML-DSA / Falcon public keys),

can compute their own ECR′(W, O) without any privileged cooperation from ERES. The discrepancy Δ \= |ECR − ECR′| is itself a measurable error term, and a non-zero Δ is a falsification event — either an ERES reporting error, an auditor methodology error, or a substrate compromise. All three are actionable.

## **6.2 The repeatable-drift detector**

A particularly important class of failure that ECR detects: repeatable drift. If across multiple windows W₁, W₂, …, Wₙ the seven-vector ⟨ECR₁…ECR₇⟩ trends monotonically downward in any dimension without an explanatory event, the system is leaking erasure in a structured way — quietly accumulating residual permissions or stale attestations. This is the AI-determinism failure mode (deterministic but drifting) reframed in governance terms, and it is distinct from ordinary random error. ECR makes it visible.

## **6.3 The narrative-system exclusion**

A pure narrative system cannot produce an ECR because it has no enumerable required\_erasure population. The denominator does not exist. Any system that publishes an ECR (with the seven-vector and the instrument provenance) has, by that publication alone, exited the narrative-system category. ECR is therefore the bright line.

# **8\. ECR and Proof-of-Resonance**

Meritcoin operates under Proof-of-Resonance, not Proof-of-Work. The distinction matters for ECR specifically:

### **Proof-of-Work has no native erasure semantics**

Once a block is buried under sufficient subsequent work, it is buried. There is no operational notion of “retiring” that block — the chain treats it as permanent. An ECR-equivalent computed against a PoW substrate would always be near zero in the denominator-defined sense (almost nothing is ever required to be erased) and therefore vacuous.

### **Proof-of-Resonance is continuously revisable**

Resonance state is a tuning, not a settlement. The four BERA Resonance Metrics — ARI (Aggregate Resonance Index), ERI (Ethical Resonance Index), RHC (Resonant Harmony Cycle), RCI (Relational Coherence Index) — are re-evaluated at every Meritcoin update window. A prior tuning that no longer holds is not buried; it is retired, and the retirement is exactly what ECR measures. “It’s not mining — it’s tuning” is therefore not a slogan but a structural claim about the erasure surface.

ECR is the metric that audits whether the tuning system is honoring its own retirement obligations. Without ECR, Proof-of-Resonance would have the same auditability gap as Proof-of-Work; with ECR, it has the closure property that PoW lacks by construction.
